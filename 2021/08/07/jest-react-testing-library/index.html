简单入门Jest和React-testing-library

Jest和React-testing-library用于单元测试,本文主要基于React框架介绍。

Let’s begin

首先先创建React项目,终端输入npx create-react-app testing-react-app,安装相应库到devDependencies中npm i --save-dev @testing-library/react react-test-renderer,然后在src目录下创建一个components文件夹,在其中创建todo.js,写入如下代码:

import React from 'react'

const todo = ({todo}) => {
  const {id,title,completed} = todo
  const h1 = <h1>{title}</h1>
  const text = completed ? <strike>{h1}</strike> : h1;

  return (
    <div data-testid={`todo-${id}`}>
      {text}
    </div>
  )
}

export default todo

然后在根目录的App.js写入以下代码

import './App.css';
import Todo from './components/todo'

function App() {
  const todos = [
    {id:1,title:'wash dishes',completed:false,},
    {id:2,title:'make dinner',completed:true,},
  ]
  return (
    <div className="App">
      {todos.map((todo)=>{
        return (<Todo todo={todo}/>)
      })}
    </div>
  );
}

export default App;

现在我们yarn start,在打开的窗口观察现象,第一行正常显示wash dishes,第二行则有删除线显示make dinner,这是<strike>的作用

OK,现在开始测试用例的编写

在components文件夹下创建__test__文件夹,写入todo.test.js,这里其实就是写入和你要测试的组件名称相同,其实还有一种方法是在根目录创建一个test文件夹,然后与其他文件夹一一对应。

  • 首先引入相应的测试库及Todo组件
    import { render, screen, cleanup } from "@testing-library/react"
import renderer from 'react-test-renderer'
import Todo from "../todo"
```  
每个测试点可写为以下这种格式
````js
test('测试名称',()=>{
    expect(被测试组件).method(预期结果)
    ...
})
````
比如我现在测试第一个todo
```JS
test('should render non-completed component', () => {
    const todo = {id:1,title:'wash dishes',completed:false}
    // expect(true).toBe(true)
    render( < Todo todo={todo}/ > )
    const todoElement = screen.getByTestId('todo-1')
    expect(todoElement).toBeInTheDocument()
    expect(todoElement).toHaveTextContent('wash dishes')
    expect(todoElement).not.toContainHTML('<strike>')
})
    可以看到,用引入的render渲染一个Todo组件,这个组件的prop与真实App.js中的todo相同,通过screen.getByTestId(id)拿到这个Element,这要求之前的todo.js文件中DOM元素中有data-testid这个attribute,接下来三行toBeInTheDocument()表示这个渲染在当前Document中,toHaveTextContent('wash dishes')表示有这个内容,.not.toContainHTML('<strike>')表示不包含<strike>,yarn test测试显示成功.
  • 接下来测试第二个todo
    接着写入以下代码
    test('should render completed component', () => {
    const todo = {id:2,title:'make dinner',completed:true}
    // expect(true).toBe(true)
    render( < Todo todo={todo}/ > )
    const todoElement = screen.getByTestId('todo-2')
    expect(todoElement).toBeInTheDocument()
    expect(todoElement).toHaveTextContent('make dinner')
    // expect(todoElement).toContainHTML('<strike>')   idk why it's failing
    // expect(todoElement).toContainHTML('</strike>') no i think it's wrong
    expect(todoElement.innerHTML.includes('<strike>')).toBeTruthy()
})
    之前的都一样,直接看最后三行,首先前两行我感到很迷茫,expect(todoElement).toContainHTML('<strike>')这个按理来说应该能通过测试,但却不行,在错误提示中显示现在的结果是完整的DOM结构,而expect的只有<strike>,所以出错,于是我改成了第二行,pass了,但是我多了个心眼在上一个测试案例中也用了这个,结果也pass了,按理来说它是没有这个的,所以我觉得这个不靠谱,改成最后一行是正确的写法.
  • wait,what’s snapshot?
    snapshot意为快照,我觉得好像就是把这个内容拍下来(打印出来)?
    看看代码:
    test('match snapshot',()=>{
  const todo = {id:1,title:'wash dishes',completed:false}
  const tree = renderer.create(<Todo todo={todo}/>).toJSON()
  // console.log(tree);
  expect(tree).toMatchSnapshot()
})
    之前引入的renderer派上了用场,create方法内写入DOM元素,创建这个节点,紧接着toJSON()转化为JSON格式,最后expect(tree).toMatchSnapshot(),这时我们yarn test,这个测试用例会通过,然后创建一个文件夹(__snapshots__),里面的文件(todo.test.js.snap)如下:
    // Jest Snapshot v1, https://goo.gl/fbAQLP

exports[`match snapshot 1`] = `
<div
  data-testid="todo-1"
>
  <h1>
    wash dishes
  </h1>
</div>
`;
    其实第一次无论怎样都会通过,但如果这是你做些变化,比如在todo.js文件中wash前加Hello,就通过不了,但它会提示你更新,点击u,pass后看该snap文件发现其中的变化。
  • 你以为这样就完了?
    细心的你或许发现引入的cleanup没有使用,是的,望文生义的话,它就是清除的意思,Jest提供的全局方法afterEach,以为每个测试后执行其中内容,因此,我们最好写入:
    afterEach(()=>{
  cleanup()
})
    这样每个测试后都会cleanup

    以上就是单元测试简单入门,还有很多内容等着好学的你我去慢慢学哦
